申长硕PB22020518 & 滕润喆PB22061212

引言

开发意义

随着信息技术的不断进步，越来越多的学校开始建设信息化平台。 作为英语俱乐部的往届的骨干，结合课程特性创建一个英语俱乐部交流平台尤为重要。
由于参与者和活动数量庞大，手工管理俱乐部的交流和活动变得越来越不切实际。 利用计算机技术进行现代信息管理可以显著提高这些操作的准确性和效率。

可行性

传统的俱乐部交流和活动管理常常存在准确性低、效率低等问题。
计算机具有存储快、检索方便、准确性高的特点，可以有效解决这些问题，使管理过程更加顺畅和可靠。

应用环境
  • 架构： B\S (Browser/Server) 架构
  • 数据库： mysql Ver 14.14 Distrib 5.7.24, for Linux (x86_64) using EditLine wrapper
  • 技术路线： 使用 Django 框架
  • 开发工具： 使用 PyCharm 作为主要的开发环境，结合 Git 进行版本控制和协作开发
  • 辅助技术： 使用爬虫技术，获取初始的寻宝体验的谜题
获取源码

所有程序已托管到 `github` 平台，可以直接利用如下命令获取：

git clone git@github.com:Stephen-Shenc/USTC-English-Club.git

请确保配置相关数据库后才可执行。

系统总体设计

功能模块

用户模块
账户管理
  • 注册： 用户可以通过填写邮箱和密码完成注册，并通过邮箱验证确认账户。
  • 登录： 用户可以使用注册时填写的邮箱和密码进行登录，系统会管理用户的会话状态。
  • 密码重置： 用户可以通过提供注册时使用的邮箱地址，收到的邮件中的验证链接来重置密码。
  • 基本信息编辑： 用户可以更新个人的基本信息，包括用户名、个人简介等。
参与抽奖
  • 抽奖： 输入参与人数进行对应数量的抽奖。
体验寻宝
  • 全部浏览： 用户可以浏览所有寻宝内容。
  • 按难度分类浏览： 用户可以按照难度级别筛选寻宝内容。
  • 详情页面查看答案： 用户可以在详情页面查看每个寻宝的答案。
参与论坛
  • 发帖： 用户可以在论坛中发布帖子。
  • 评论： 用户可以对帖子进行评论。
  • 二级评论： 用户可以对其他评论进行回复。
  • 点赞： 用户可以点赞帖子和评论。
  • 分类查看： 用户可以按主题分类浏览帖子。
管理员模块
数据统筹
  • DashBoard： 可以查看目前已有的用户、帖子、评论等的数量。
用户管理
  • 用户信息查看： 可以查看用户密码以外的所有信息。
  • 用户删除： 可以直接删除不合规用户。
抽奖管理
  • 奖品查看： 查看奖品池中的奖品类别及个数。
  • 奖品管理： 管理抽奖池中的奖品类别及其数量。
论坛管理
  • 分类查看帖子： 可以按照类别查看所有帖子。
  • 查看帖子详情页： 从“普通用户”视角看帖子，且拥有普通用户的权限。
  • 管理帖子： 可以审核或删除用户发布的帖子。
  • 管理评论： 可以审核或删除用户发布的评论。
寻宝管理
  • 查看Official Riddles： 管理员可以查看从官方获取的Riddles。
  • 删除Official Riddles： 管理员可以删除不需要的Official Riddles。
  • 添加Official Riddles： 管理员可以添加新的Official Riddles。

实现与测试

游客视角
1、访问主页
2、访问论坛、可以看各种类别的帖子
3、点击已有的评论可以弹出回复框，进行回复
4、如果没有登陆时进行评论/回复/发帖，都会跳转到登陆页面
5、访问寻宝体验页面，查看各种Riddles
6、Riddle详情页、可以点击show answer获取答案
7、尝试访问Admin Panel管理员面板，会显示没有权限，然后5秒返回
8、可以查看抽奖页面，但是没有抽奖权限
普通用户视角（不再重复上述的游客视角）

帐号管理部分
1、填写用户名、邮箱、密码点击注册
2、可以收到来自ustc_englishclub@163.com的验证码
3、之后弹出输入框提示输入验证码，同时窗口上方有提示信息
4、验证成功后可以进行登陆
5、注册成功跳转到用户信息界面，可以查看当前信息
6、还可以修改部分信息，之后点击下方change即可完成修改
7、如果在登陆时忘记密码，也可以点击下方忘记密码按钮
8、弹出输入框，可以使用邮箱验证码进行密码更改
9、验证码验证通过之后即可完成密码修改
10、之后便可以正常登陆

论坛部分
1、查看按主题分类的各个论坛
2、点击浏览发帖的详情界面
3、点击其中的大拇指可以点赞
4、翻到底部有一个评论按钮，可以用于做一级评论
5、点击下方 "leave a comment/reply" 即可进行评论
6、再点击任意一个已经有的评论，则会弹出回复框
7、输入内容并发送，则完成了对一级评论的回复功能
8、还可以点击Forum_index中的share，跳转到分享界面
9、填写标题，选择主题，有编辑助手，还可上传图片文件
10、提交之后可以直接跳转到对应详情页，之后可以执行和前面一样的评论之类的操作

寻宝体验部分
1、查看按难度分类的各个论坛
2、点击浏览Riddle的详情界面
3、可以点击show answer查看答案
4、关于点赞、评论等功能就不再重复实现了，都是一样的

抽奖部分
1、输入参与人数，可以抽取对应个数的奖品
2、抽奖的逻辑是中间的img旋转3s之后提交表单（提升用户体验），后端按照奖品数量为权值进行抽奖，最后返回抽取到的奖品

管理员面板——无权限，同游客一样5秒返回
管理员视角
管理员拥有普通用户的所有权限和功能，下面只展示管理员特有的权限——数据管理
1、登陆专门注册的管理员账号super@mail获取管理员权限
2、进入Admin_Panel，首先看到的是总数据，这些数据都是自编脚本做的“虚拟数据”
3、点击Posts中的任意一类，可以跳转到对应类别的发帖（在此处展示的都是没有解析的样式）
4、点击view，可以以正常用户视角进行查看，当然也可以进行点赞和评论
5、点击删除，首先弹出是否确定（所有的删除操作均经过此逻辑）
6、图示相比左边删除了id为1714的评论
7、还可以展示所有的评论，不过此时只赋予删除操作
8、图示删除了id为329的评论
9、还可查看所有用户的基本信息
10、图示删除了id为7的用户
11、管理员还有权限查看已有奖品及数量信息
12、可以直接修改现有奖品的数量信息
13、还可以添加奖品信息
14、图示添加了5个小风扇奖品
